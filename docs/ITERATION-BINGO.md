## 一、问题发现

### 1.1 用户反馈

豆奶在使用 DouNai v0.1（传统任务列表模式）的过程中发现：

> "总是懒得去新建任务并点击完成"

这揭示了 v0.1 核心循环的一个根本问题：**两步都需要意志力**。

| 步骤 | 行为 | 意志力消耗 |
|------|------|-----------|
| 第一步 | 新建任务 | 需要主动思考"今天要做什么" |
| 第二步 | 点击完成 | 需要主动回来记录 |

对于拖延症用户来说，这两步本身就是新的拖延对象——工具变成了负担。

### 1.2 灵感来源

豆奶在社交媒体上看到了 **Bingo 卡片式任务管理** 的玩法：

- "ADHD生活bingo"：5×5 网格，包含日常任务（规律睡眠、扫地、记账、运动15min等）
- "维持生命体征Bingo"：极低门槛版（"睡着了""醒来睁眼""发呆"也算完成），底部写着"无论完成多少，又活了一天，真棒！"

**核心洞察**：用"消消乐"的方式随机展示任务，通过连线规则给予奖励反馈，将"完成任务"从苦差事变成游戏。

## 二、Bingo 模式 vs 传统列表

### 2.1 为什么 Bingo 比 Todo List 更适合

| 维度 | 传统列表 | Bingo 卡片 |
|------|---------|-----------|
| 决策成本 | 高（要自己新建） | 低（系统随机生成） |
| 完成动力 | 单个勾选，满足感弱 | "差一个就连线了"的驱动力 |
| 反馈强度 | ✅ 一个勾 | 消除动效 + 连线奖励 |
| 失败压力 | "还有X个没完成" | 格子本身就是选择题 |
| 自由度 | 完全自主（反而瘫痪） | 有限选择（降低决策疲劳） |

### 2.2 行为科学支撑

- **选择悖论**（Schwartz, 2004）：选项过多导致决策瘫痪。Bingo 把"无限可能"缩减为 25 个具体格子
- **目标梯度效应**（Goal Gradient Effect）：越接近目标行动越快。"还差一个就连线"比"还有5个任务"更有驱动力
- **可变比率强化**：每天随机刷新的卡片 + 不同连线组合 = 持续新鲜感

### 3.2 核心交互流程

#### 每日刷新
1. 每天 00:00 自动从**任务池**随机抽取 25 个任务填充 5×5 网格
2. 任务池 = 用户自建任务 + 系统预设通用任务（兜底）
3. 中心格可设为 FREE（自动完成），降低连线门槛

#### 完成任务
1. 点击格子 → 翻转/消除动效 → 标记完成
2. 检测是否形成连线（横/竖/对角线）
3. 连线形成 → 整行高亮 + 庆祝动效 + 额外 🌸 奖励

#### 拖拽替换
1. 用户通过顶部输入框新建任务 → 出现在"待替换区"
2. 长按新任务方块 → 拖拽到 Bingo 网格中想替换的格子
3. 被替换的任务回到任务池（不删除），下次刷新可能再出现

### 3.3 积分系统适配

| 行为 | 获得 |
|------|------|
| 完成单个格子 | 无直接积分（已嵌入连线激励） |
| 完成一条连线（5格） | +1 🌸 |
| 完成 2 条连线 | +2 🌸 |
| 完成全部 12 条连线 | +12 🌸 + 额外 bonus |
| 日记评估 | 保持不变（最多 +2.5 🌸） |
| 连续天数 | 保持不变（3天+1, 7天+3） |

> 注：积分规则待进一步讨论确定。

### 3.4 任务池设计

#### 用户自建任务
- 通过顶部输入框添加，永久保存在任务池
- 可在设置页管理（删除/编辑）

#### 系统预设任务（兜底）
当用户任务池不足 25 个时，用预设任务补齐：

```
轻量级（类似"维持生命体征"）：
  喝一杯水、站起来走走、深呼吸10次、看窗外1分钟、
  伸个懒腰、整理桌面、发一条消息给朋友、听一首歌...

日常级：
  刷牙、洗脸、吃早餐、出门散步15分钟、
  看10页书、写3句话的日记、整理一个抽屉...
```
